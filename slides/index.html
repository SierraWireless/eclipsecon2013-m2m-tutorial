Serial port API


-- open the serial port
local serial = require("serial")
local portconfig = { parity="none", flowControl="none",
                     numDataBits=8, numStopBits=1, baudRate=115200 }
local sdev = serial.open("/dev/ttyS0", portconfig)

-- Now use it!
-- the provided API is aligned with file manipulation API
local read_data = sdev:read(2) -- reads 2 bytes on serial port

sdev:write("somedata") -- writes buffer to serial port

Modbus

  • Industrial communication protocol
    De facto standard created by Modicon/Schneider Electric
  • Application layer protocol
    stateless client-server protocol
  • Transactions
    • request by a client (= master)
    • response by the server (= slave)

Modbus

  • Serial or TCP
    Many devices can be connected in the same network
  • 2 transmission mode: 'RTU', 'ASCII'
    Each mode comes with specific encoding, framing and error checksum mecanism
  • Manipulates raw data (bits or words)
    Data type names are related to PLC/relays context: coil, discrete input, ...

Modbus Lua API

Initializing the client protocol stack


-- load the module
local modbus = require("modbus")

-- As Modbus module uses Serial module,
-- you can use the same port configuration table
local portcfg = { parity="none", flowControl="none", numDataBits=8,
		  numStopBits=1, baudRate=115200 }

-- create new modbus instance using serial configuration
-- and selecting modbus transmission mode.
local mdev = modbus.new("/dev/ttyS0", portcfg, "ASCII")

Modbus Lua API

Sending / Reading data


-- reads 2 registers (16 bits word) on slave id 10 at address 42.
local input_buffer = mdev:readHoldingRegisters(10, 42, 2)

-- decode input buffer: e.g. 2 signed short int
-- (decoding depends on data mapping on requested modbus device)
local shortint1, shortint2 = string.unpack(input_buffer, "h2")

-- writes to device id 1, address 90, 3 unsigned short values
mdev:writeMultipleRegisters(1, 90, string.pack("H3",21,159,357))

-- Many other way to read/write data to a Modbus device!

Data flow

Three main steps:

  1. Data is generated/acquired in asset application
  2. Data is pushed to Mihini agent using Racon lib
  3. Data is pushed to M2M Server using M3DA protocol

Lua Library initialization


-- load the module
local racon = require("racon")
-- create an `asset`:
-- software element exchanging data with M2M server,
-- identified by unique id
local house = racon.newasset("house")

Pushing data


-- push data to Mihini agent:
-- timestamped temperature of the bedroom
house:pushdata("bedroom", { temp=42, timestamp=os.time()}, "now")
-- at this point the data is in Mihini agent.
-- `hourly` is the data queue in the agent where the data is stored
-- each queue is linked to a data policy:
-- this is how / when the data is *automatically* pushed to server

-- send data from agent to M2M server at a slower pace:
house:pushdata("bedroom", { temp=42, timestamp=os.time()}, "hourly")

-- for development purpose (or debug),
-- you may want to force data flushing from agent to server
-- racon.triggerpolicy("hourly")
